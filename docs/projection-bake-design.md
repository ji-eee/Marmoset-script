# 리버스 프로젝션 베이크 설계

정면·후면 캡처 2장을 메시 UV로 되쏘아(gather) 텍스처를 만드는 방식.
스크린→UV로 흩뿌리는(scatter) 대신, UV 텍셀마다 3D 위치를 구해 캡처에서 색을 가져온다(gather).
gather라야 UV 공간을 빈틈없이 채우고 오클루전·마스킹을 텍셀 단위로 제어할 수 있음.

## 전체 흐름

```
정면 캡처(원래 포즈) ─┐
                      ├─ 카메라(위치·회전·FOV로 재구성)
대상 오브젝트 180도 Y 회전 → 후면 캡처 → 원복 ─┘
        │
        ├─ 정면/후면 카메라뷰 깊이 버퍼(z-buffer) 각각 생성
        │
        └─ 메테리얼 그룹마다: UV 삼각형 래스터화 → 텍셀마다
             (월드 위치 P, 노멀 N 보간) → 정/후/옆 분류 → 투영·오클루전·샘플 → PNG
```

## 좌표계·투영

- 월드: 오른손, Y-up. 카메라는 로컬 -Z를 바라봄, 로컬 +Y 위, +X 오른쪽
- Euler 규약은 `EULER_ORDER="YXZ"` 한 곳에서 정의(=`Ry@Rx@Rz`). 자세한 배경은 api-notes 참고
- 카메라 월드행렬 `W = T(pos)·R(rot)`. 뷰행렬은 강체 역행렬 `R^T·T(-pos)`
  - 카메라에 스케일이 없다는 전제(카메라는 스케일 안 씀)
- 투영: 세로 FOV로 `tan(fovv/2)`, 가로는 `aspect·tan(fovv/2)`
  - 카메라 공간 `(xc,yc,zc)`, -Z를 보므로 앞쪽은 `zc<0`, `depth=-zc>0`
  - `ndc_x=(xc/depth)/tan_h`, `ndc_y=(yc/depth)/tan_v`
  - 픽셀: `px=(ndc_x·0.5+0.5)·W`, `py=(0.5-ndc_y·0.5)·H` (이미지 y는 위→아래라 뒤집음)
- `Camera.project(P) -> (px, py, depth, in_front)`

## 텍셀 → 3D (UV 삼각형 래스터화)

- UV를 텍셀 좌표로: `tx=u·size`, `ty=(1-v)·size` (기본 `flip_v=True`, v-up 가정)
- 삼각형의 UV 바운딩 박스 안 텍셀 중심마다 edge-function 무게중심 `(w0,w1,w2)` 계산
- 삼각형은 평면이라 UV에서 구한 무게중심이 3D에서도 그대로 성립(affine)
  → `P = w0·p0 + w1·p1 + w2·p2`, `N = normalize(w0·n0+...)`가 정확. 원근 보정 불필요
- UV 겹침(같은 텍셀을 여러 삼각형이 씀)은 facing 값이 더 정면인 샘플이 이기도록 score 버퍼로 관리

## 정면 / 후면 / 옆면 분류

- `vcam` = P에서 카메라로 향하는 단위벡터(직교면 카메라 forward의 반대)
- `facing_front = dot(N, vcam)` — 1에 가까울수록 정면
- `cos_thresh = cos(SideMaskAngle)`
- `facing_front >= cos_thresh` → **정면**: 정면 캡처에서 샘플
- 아니면 후면 후보: 회전 포즈 `P'=R_C·P`, `N'=회전(N)`로 `facing_back` 계산
  - `facing_back >= cos_thresh` → **후면**: 후면 캡처에서 샘플
  - 그 외(정면·후면 모두 임계 미만인 스치는 각도) → **옆면**: 투명으로 남김(마스킹)
- 이 방식이 요구사항의 "밀리는 옆면은 마스킹"을 그대로 구현

## 오클루전 (겹친 부분·뒤쪽 안 묻기)

캡처와 같은 해상도로 두 가지 버퍼를 함께 만든다(모든 가시 삼각형 래스터화).

- 깊이 버퍼(z): 픽셀별 최근접 depth. 원근 카메라는 `1/depth` 보간(원근 보정)
- 오브젝트 ID 버퍼: 픽셀별 최근접 표면이 속한 오브젝트 id(`bake_id`). 없으면 -1
- 정면 포즈용·후면 포즈용 각각 1쌍(깊이+ID)

텍셀 판정(`_visible`):
- 그 픽셀의 최근접 표면이 **다른 오브젝트**면(ID 불일치) → 스킵
- 같은 오브젝트인데 `depth > near + (bias_abs + bias_rel·depth)`면(자기 앞면에 가림) → 스킵
- 추가 안전장치: 투영 지점이 배경(캡처 알파≈0)에 떨어지면 색을 안 칠함

실제 파이프라인은 **오브젝트별 격리 캡처**(아래 참고)라, 한 번의 베이크에는 그 오브젝트만
들어감 → ID 버퍼는 단일 id가 되어 크로스오브젝트 거부는 자연히 안 일어나고, 깊이 버퍼가
자기 겹침만 처리. 깃털이 머리에 묻던 문제는 캡처 단계에서 이미 분리돼 해결됨.

## 샘플링 (테두리 검은 번짐 방지)

캡처를 바이리니어로 뽑을 때 **알파 가중** 방식 사용(`sample_bilinear_weighted`).
`rgb = Σ(rgb·a·w) / Σ(a·w)`, `alpha = Σ(a·w)`.
실루엣 경계에서 투명 배경(rgb=0)이 섞여 색이 검게 죽는 것을 막음.
반환 alpha는 배경 판정(`require_opaque_alpha`)에 그대로 사용.

## 오브젝트별 격리 캡처 + 합성

여러 오브젝트가 겹치면(예: 슬롯 사이의 깃털) 한 장의 스크린샷으로는 뒤 오브젝트의
가려진 면이 안 찍힌다. 그래서 오브젝트마다:

1. 나머지 대상 메시를 잠깐 숨김(`visible=False`, 조상은 제외) → 그 오브젝트만 보이게
2. 정면 캡처
3. 그 오브젝트를 자기 bounds 중심 기준 180도 턴테이블 → 후면 캡처 → 즉시 원복
4. 숨겼던 오브젝트 가시성 원복
5. 그 오브젝트 하나만으로 베이크(자기 깊이 버퍼 = 자기 겹침만)

이렇게 하면 앞 오브젝트에 가려 비던 뒷면이 채워지고, 다른 오브젝트 색이 묻지도 않음.
마지막에 메테리얼별로 각 오브젝트 결과를 알파 최대값 기준으로 **합성**
(`composite_max_alpha`). 보통 오브젝트별 UV 아일랜드가 겹치지 않아 각자 영역만 채움.

## 결과 2장 (variants)

오브젝트마다 두 variant를 뽑고(`bake_variants`), 합성 후 메테리얼별로 2장 저장.

- `masked`: Side Mask Angle 적용(옆면 투명) → 합성 후 테두리 블러(최종용)
- `full`: `unmasked` 모드 — 옆면 마스킹·오클루전·배경 판정 전부 끄고, 텍셀마다
  앞/뒤 캡처 중 더 정면인 쪽의 색을 바름(전후면 스미어). 합성 후
  `fill_transparent`로 남은 투명 텍셀을 최근접 색으로 채워 **이미지 전체를 완전
  불투명**으로 만듦

`fill_transparent`는 칠해진 텍셀 전체를 시드로 하는 멀티소스 BFS로, 각 투명 텍셀에
가장 가까운 색을 복사하고 알파를 255로 강제한다. 무한 UV 패딩과 같아서, 모델에
적용했을 때 UV 시임 경계에 배경(흰색)이 비쳐 "테두리가 어긋나 보이는" 문제도 함께
사라진다. 이전 구현에서 `_full`에 남던 실루엣/겹침 부근의 투명 구멍이 그 흰 테두리의
원인이었음.

## 테두리 블러 (edge blur, postprocess.py)

경계만 부드럽게, 내부 디테일은 유지.

- 알파를 박스 블러 → 불투명↔투명 전이가 완만해짐(페더링)
- 색은 알파 가중 박스 블러(`blur(rgb·a)/blur(a)`)로 → 검은 헤일로 없이 색이 경계 밖으로 몇 px 번짐(패딩 효과)
- 완전 내부 텍셀(불투명 + 이웃도 전부 불투명)은 원본 그대로 유지
- 분리형 prefix-sum 박스 블러(가로→세로), float32 버퍼, 채널 순차 처리로 4096에서도 메모리 절약

## 턴테이블 (후면 캡처용 물리 회전)

- 회전 중심 C = **그 오브젝트**의 월드 AABB 중심(자기 자리에서 뒤집힘 → 프레이밍 유지)
- 월드 변환 `R_C = T(C)·Ry(180)·T(-C)`
- 격리 캡처이므로 그 오브젝트 하나만 회전
- YXZ 규약에서 아래가 `R_C·W`와 정확히 일치(스케일·피벗 포함 증명됨)
  - `pos' = C + Ry180(pos + pivot - C) - pivot`,  `Ry180(x,y,z)=(-x,y,-z)`
  - `rot' = [rx, ry+180, rz]` (Y가 바깥 축이라 월드 Y 프리멀티플과 동일)
  - scale·pivot 그대로
- 베이크의 후면 분기도 같은 `R_C`를 월드 좌표에 적용하므로 캡처와 정확히 맞물림
- 회전 전 원본 transform 스냅샷 → 캡처 후 `finally`에서 원복(예외가 나도 씬 보존)

## 메테리얼 그룹 → PNG

- 각 `MeshObject`의 `SubMeshObject`로 (메테리얼명, 삼각형 인덱스 범위) 수집
- 다중 메테리얼 메시는 메테리얼별 "뷰"로 쪼개, 해당 범위 삼각형만 그 그룹에서 베이크
- 그룹마다 `ImageRGBA(size,size)` → `<씬>_<메테리얼>.png` 저장

## 성능

- numpy 없이 순수 파이썬. 비용 ≈ O(삼각형 수 + 커버된 픽셀/텍셀 수)
- 캡처 PNG 디코드(어댑티브 필터 언필터)와 4096 텍셀 순회가 느릴 수 있음
- 권장: 우선 1024~2048로 확인 후 필요하면 4096. 렌더용 메시 폴리곤이 과하면 시간이 늘어남
- 여지: numpy 있으면 삼각형 단위 벡터화, Marmoset 깊이 패스(GPU)로 오클루전 대체 등(향후)
